Modèles de données

Schema Prisma complet

// ── ÉTABLISSEMENT ─────────────────────────────────────
model Etablissement {
  id          String    @id @default(cuid())
  nom         String
  region      String
  filieres    Filiere[]
  annees      AnneeFormation[]
  createdAt   DateTime  @default(now())
}

// ── ANNÉE DE FORMATION ─────────────────────────────────
model AnneeFormation {
  id              String    @id @default(cuid())
  label           String    // ex: "2025/2026"
  etablissementId String
  etablissement   Etablissement   @relation(...)
  groupes         Groupe[]
  calendriers     Calendrier[]
}

// ── FILIÈRE ────────────────────────────────────────────
model Filiere {
  id              String    @id @default(cuid())
  code            String    // ex: "EB" (Électricité Bâtiment)
  nom             String    // ex: "Électricité de Bâtiment"
  etablissementId String
  modules         Module[]
  groupes         Groupe[]
}

// ── MODULE ─────────────────────────────────────────────
model Module {
  id              String    @id @default(cuid())
  code            String    // ex: "M101", "M102"
  intitule        String    // ex: "Métier et formation"
  mhGlobale       Int       // Masse horaire globale (MH.G)
  filiereId       String
  niveau          NiveauEnum
  avancements     Avancement[]
  notes           NoteModule[]
}

// ── GROUPE ─────────────────────────────────────────────
model Groupe {
  id              String    @id @default(cuid())
  code            String    // ex: "EB101"
  annee           Int       // 1 ou 2 (extrait du code)
  mode            ModeEnum  // R-PP, R-FF, FPA-PP, FPA-FF
  filiereId       String
  anneeFormId     String
  statut          String    // "Actif" | "Clôturé"
  stagiaires      Stagiaire[]
  avancements     Avancement[]
}

// ── STAGIAIRE ──────────────────────────────────────────
model Stagiaire {
  id              String    @id @default(cuid())
  cef             String    @unique  // ex: "2009052100036"
  nom             String
  prenom          String
  groupeId        String
  notes           NoteModule[]
}

// ── AVANCEMENT ─────────────────────────────────────────
model Avancement {
  id              String    @id @default(cuid())
  groupeId        String
  moduleId        String
  mhGlobale       Float     // MH.G
  mhPlanAF        Float     // MH.P AF
  mhRealise       Float?    // Réal.G (null = non démarré)
  tauxReel        Float?    // Calculé: Réal.G / MH.G
  nbSeancesVal    Int       // NB.SValidées
  nbSeancesEnCours Int      // NB.SEn cours
  sourceFile      String    // Nom du fichier source
  importedAt      DateTime
}

// ── NOTE MODULE ────────────────────────────────────────
model NoteModule {
  id              String    @id @default(cuid())
  stagiaireId     String
  moduleId        String
  cc              Float     // Contrôle Continu /20
  efm             Float     // Examen Fin Module /40 (0 si absent)
  efmStatut       EFMStatut // PRESENT | ABSENT
  moyenneOff      Float     // (CC + EFM) / 3
  moyenneNorm     Float     // (CC + EFM/2) / 2
  sourceFile      String
  importedAt      DateTime
}

// ── CALENDRIER ─────────────────────────────────────────
model Calendrier {
  id              String    @id @default(cuid())
  anneeFormId     String
  typeCalendrier  String    // "1A-CDJ R-PP", "2A-CDJ R-FF", etc.
  totalJours      Int       // Nombre de "1" dans toute la ligne
  joursRealises   Int       // "1" jusqu'à date_reference
  tauxTheorique   Float     // joursRealises / totalJours
  dateReference   DateTime
  jourDetails     Json      // [{date, statut: "1"|"ST"|"holiday"|"EFF"}]
  sourceFile      String
}

// ── ENUMS ──────────────────────────────────────────────
enum NiveauEnum  { S Q T TS }
enum ModeEnum    { R_PP R_FF FPA_PP FPA_FF }
enum EFMStatut   { PRESENT ABSENT }
04

Parsers de fichiers

Parser A — État d'avancement (Excel)

Fichier source : EtatXLS_*.xlsx — Sheet : Sheet1

Colonnes attendues (row 0 = headers)

  • Groupe — Code groupe (ex: EB101)
  • Statut — "Actif" ou autre
  • Module — Intitulé long du module
  • MH.G — Masse horaire globale (Float)
  • Réal.G — Heures réalisées (Float | NaN)
  • % Gl. — Taux réel brut (Float)
  • NB.SValidées — Séances validées (Int)
  • NB.SEn cours — Séances en cours (Int)
⚠
Réal.G peut être NaN (module non démarré) — traiter comme 0 pour affichage, conserver null en base. Un taux % Gl. > 1.07 (107%) = anomalie à signaler.

Parser B — Calendrier (Excel)

Fichier source : CALENDRIER_25-26.xlsx — Sheet : 25-26

Structure de la feuille

  • Row 2 (index 2) : en-têtes de semaines (numérotées 1–45)
  • Row 3 (index 3) : jours de la semaine (L, M, M, J, V, S)
  • Row 4 (index 4) : dates réelles (datetime objects)
  • Rows 5+ : lignes de types de calendrier avec cellules "1", "ST", texte arabe, NaN

Lignes à extraire uniquement

  • 1A-CDJ (Résidentiel-passage) — pour groupes EB1xx
  • 2A-CDJ (Résidentiel-Fin de formation) — pour groupes EB2xx
Valeur cellule Signification Compter ? Statut stocké
"1" ou 1Jour de formation valide✅ Oui"FORMATION"
NaN / videWeek-end / non applicable❌ Non"VIDE"
Texte arabeJour férié national❌ Non"FERIE"
"ST"Période de stage❌ Non"STAGE"
"EFF"Fin de formation❌ Non"FIN"
Texte "عطلة..."Vacances scolaires❌ Non"VACANCES"

Parser C — PV EFM (PDF)

Fichiers source : PV_EFM_EB101_M*.pdf

Données extraites par regex du texte PDF

  • En-tête : Établissement, Filière, Année, Groupe, Niveau, Intitulé Module
  • Statistiques : Inscrits, Présents, Absents
  • Tableau : CEF (14 chiffres), Nom & Prénom, CC/20, EFM/40, Moy Module/20
  • Cas Absent : valeur EFM = texte "Absent" → stocker efm=0, efmStatut=ABSENT
// Regex patterns pour extraction PDF

// Extraction header
ETABLISSEMENT: /Etablissement\s*:\s*(.+)/i
FILIERE:       /Filière\s*:\s*(.+)/i
ANNEE:         /Année de formation\s*:\s*(\d{4}\/\d{4})/i
GROUPE:        /Groupe de formation\s*:\s*(EB\d{3})/i
MODULE:        /Intitulé du Module:\s*(.+?)\s*\(M(\d+)\)/i
INSCRITS:      /Inscrits:(\d+)\s*\|Présents:(\d+)\s*\|Absents:(\d+)/i

// Extraction lignes tableau stagiaires
ROW: /^(\d{13,14})\s+(.+?)\s+([\d.]+)\s+([\d.]+|Absent)\s+([\d.]+)$/m

// Validation CEF (14 chiffres)
CEF: /^\d{13,14}$/

Pipeline d'import

UPLOAD
→
PARSING
→
NORMALISATION
→
VALIDATION (Zod)
→
STOCKAGE
🔒
Règle critique : Les données sources ne sont jamais modifiées. Toutes les transformations sont traçables via sourceFile + importedAt sur chaque enregistrement. Les valeurs originales sont conservées telles quelles.

Étapes détaillées

1. Upload — Réception fichier via POST /api/import, vérification type MIME (xlsx, pdf), taille max 10MB
2. Parsing — Sélection parser selon type (SheetJS pour xlsx, pdf-parse pour PDF), extraction des données brutes
3. Normalisation — Conversion types, gestion NaN/null, calcul champs dérivés (tauxReel, moyennes), detection statuts
4. Validation Zod — Vérification schémas, détection anomalies (taux > 107%), validation CEF format
5. Stockage — Upsert Prisma (conflit sur cef+moduleId pour notes, groupeId+moduleId pour avancement)
6. Log — Enregistrement ImportLog : nbLignes, nbErreurs, avertissements, durée, utilisateur
06

Calculs & formules

Taux théorique (calendrier)

Date de référence
date_ref = min(aujourd'hui, dernière_date_calendrier)
Jours réalisés
jours_realises = COUNT(cellules == "1" AND date <= date_ref)
Total jours
total_jours = COUNT(toutes cellules == "1")
Taux théorique ← Valeur principale
taux_theorique = (jours_realises / total_jours) × 100
⚑ Contrainte : 0% ≤ taux_theorique ≤ 100%

Taux réel (état d'avancement)

Taux réel brut
taux_reel_brut = (Réal.G / MH.G) × 100
Taux réel affiché
taux_reel = min(taux_reel_brut, 100)
⚑ Dépassement toléré jusqu'à 107% — au-delà = ANOMALIE signalée

Comparaison avancement

Écart
ecart = taux_reel − taux_theorique
Condition écartStatutCouleurAction
ecart > +5%En avance■ Vert—
-5% ≤ ecart ≤ +5%À jour■ Bleu—
ecart < -5%En retard■ RougeAlerte automatique

Calcul des notes (PV EFM)

Moyenne officielle OFPPT
moy_officielle = (CC + EFM) / 3
⚑ CC sur /20, EFM sur /40 — résultat sur /20
Moyenne normalisée
moy_normalisee = (CC + (EFM / 2)) / 2
⚑ Variante interne pour comparaisons — toutes valeurs ramenées à /20
⚠
Cas Absent : EFM = 0, statut ABSENT. Inclus dans les moyennes de groupe avec valeur 0. La moyenne officielle devient (CC + 0) / 3 = CC/3.

Analytique avancée

Vitesse pédagogique
vitesse = Réal.G / jours_realises (heures/jour)
Projection fin de formation
mh_restante = MH.G − Réal.G
semaines_restantes = mh_restante / (vitesse × 6 jours/semaine)
date_fin_estimee = aujourd'hui + semaines_restantes
⚑ Nécessite au moins 5 jours réalisés pour être fiable
Retard estimé
retard_jours = date_fin_estimee − date_fin_calendrier
⚑ Positif = retard, négatif = avance

Endpoints REST

Import

POST /api/import Upload + traitement fichier
Body: multipart/form-data { file, anneeFormId, type: "etat" | "calendrier" | "pv_efm" }
Returns: { success, imported: number, warnings: string[], errors: string[] }

Groupes

GET /api/groupes Liste tous les groupes actifs
Query: ?anneeFormId, ?filiereId, ?statut
Returns: Groupe[] avec taux théorique + réel calculés
GET /api/groupes/[id]/avancement Avancement par module d'un groupe
Returns: { modules: Avancement[], tauxTheorique, ecartGlobal, projection: { dateFin, retardJours } }

Stagiaires

GET /api/stagiaires/[cef]/notes Toutes les notes d'un stagiaire
Returns: { stagiaire, notes: NoteModule[], alertes: Alert[], moyenneGenerale }
GET /api/groupes/[id]/stagiaires Classement stagiaires d'un groupe
Returns: Stagiaire[] triés par moyenne, avec alertes et absences incluses

Dashboard

GET /api/dashboard KPIs globaux pour la page d'accueil
Returns: { groupesActifs, tauxMoyen, alertesCount, modulesValides, topAlerts: Alert[] }
GET /api/alertes Toutes les alertes actives
Query: ?niveau=critique|warning, ?entity=stagiaire|groupe
Returns: Alert[] { level, message, entity, entityId, createdAt }

Extensions futures

V2.0

IA prédictive

Prédiction du risque d'échec par stagiaire basée sur historique CC + assiduité. Modèle simple de régression logistique.

V2.0

Notifications

Alertes email automatiques aux formateurs dès qu'un retard pédagogique est détecté ou qu'un stagiaire passe sous 10/20.

V2.5

Export PDF

Génération de rapports pédagogiques PDF avec graphiques, par groupe ou par stagiaire, pour soumission à la direction régionale.

V3.0

Multi-établissements

Extension à tous les établissements de la Direction Régionale Oriental. Vue agrégée pour le Directeur Régional.

V2.5

Emploi du temps

Gestion du planning hebdomadaire des formateurs, intégration avec le calendrier OFPPT importé.

V2.0

Source CSV

Support de fichiers CSV en plus de Excel, avec mapping automatique des colonnes via détection d'en-têtes.

ℹ
Architecture extensible : La structure modulaire (core / import / analytics / ui) permet d'ajouter de nouvelles sources de données et indicateurs sans refactoring majeur. Chaque parser est une fonction pure sans effets de bord.
